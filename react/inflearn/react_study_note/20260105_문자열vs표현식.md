강의를 보니 textarea에 row를 row="20" 으로 작성했다.
그러나 row={20} 으로 작성해도 똑같은 결과가 나오길래, 어떤 차이가 있는지 궁금해졌다.

1. rows="20": 문자열 리터럴

<textarea rows="20" />

"" 안에 있는 값은 string으로 인식된다.
즉, React -> DOM 으로 "20" 문자열을 전달하게 된다.
HTML 속성은 내부적으로 숫자가 필요한 경우, 브라우저가 문자열을 숫자로 해석해서 동작한다.
그래서 문제 없이 20으로 처리되어 똑같이 보이는 것이다.

2. rows={20}: 자바스크립트 표현식(숫자)

<textarea rows={20} />

{} 안에 있는 값은 JS 표현식을 넣는 방식이라 number로 인식된다.
즉, React -> DOM 으로 20 숫자를 전달하게 된다.
브라우저가 속성을 처리할 때 결국 비슷하게 적용되긴 하지만, 내려보내는 값이 애초에 number이기 때문에 더 정확하다.

3. 어떤 차이가 있는데?

3-1. 타입 안정성 측면 (TS)

rows는 숫자 속성이라 타입이 보통 number로 잡힌다.
rows="20"은 TS/린트 규칙에 따라 경고가 날 수 있다.

rows={20}은 타입이 딱 일치한다.

3-2. 동적 값일 때 차이가 커짐

<textarea rows={isMobile ? 6 : 20} />

위 예시처럼 동적으로 값을 표현할 때는 무조건 {} 방식 표현이 필요하다.

3-3. 의도 전달 측면

rows="20": “문자열이네? HTML 속성이니까 그런가 보다”
→ 의도가 애매하다.

rows={20}: “숫자 속성이라 숫자를 넣었구나”
→ 의도가 명확

4. 언제 뭘 쓰는 게 맞아?

숫자/불리언 성격의 props/속성
: rows={20}, disabled={true}(혹은 disabled)

문자열 속성
: placeholder="...", name="...", id="..."

5. 결론
   rows는 숫자니까 rows={20}를 기본으로 가는 게 더 깔끔하다.
