[주기억장치 (메모리 / RAM)]
- 전원을 끄면 데이터가 사라짐
- CPU가 실행을 위해 접근하는 장치
- 필요한 주소로 바로 이동하는 무작위 접근 (Random Access) 방식을 사용
(차례로 하나씩 이동하는 순차 접근 방식과 반대)

[보조기억장치 (HDD, SSD)]
- 전원을 꺼도 데이터가 유지됨
- 보조기억장치에 있는 프로그램을 사용하려면 주기억장치로 복사를 해와서 사용함

[CPU]
- 논리, 연산작업을 하는 메인 작업 장치

[컴퓨터를 켤 때 일어나는 일들]
전원 공급 장치에 전원이 공급됨
ROM BIOS가 부트 프로그램 검사
하드웨어 검사
운영체제 로드 (보조기억장치에 있는 운영체제를 주기억장치로 가져와서 실행)
운영체제 실행

[운영체제]
예전 개발자들은 프로그램을 만들 때 하드웨어를 어떻게 사용할지에 대한 조작 프로그램까지 다 따로 만들었음
운영체제는 응용 프로그램과 하드웨어를 중개해주는 역할 (시스템 설정 등은 시스템 프로그램)
사용자가 다루는 하드웨어를 시스템에서 어떻게 사용하고 조절할지에 대한 서비스 제공
프로그래머가 입출력 장치에 신경을 덜 쓰게 해주는 고마운 존재

[컴퓨터가 2진수를 사용하는 이유]
컴퓨터 부품인 트랜지스터는 전기 신호의 켜짐(On)과 꺼짐(Off) 두 가지 상태로 작동함. 이 두 상태를 이진수의 0과 1로 표현하는 것이 가장 효율적이라서 이진수를 사용

[현대 컴퓨터의 기본 구조]
지금 사용하는 컴퓨터 구조를 직접적으로 제안한 사람의 이름은 '폰 노이만'
예전에는 전기 배선을 바꾸어 가며 프로그래밍을 했지만, 이제는 프로그램이 메모리 안에 들어간 구조가 됨
메모리에 있는 프로그램을 읽어가면서 CPU가 기능을 수행하게 됨

[이진수를 싣고 다니는 버스들]
Control bus (제어 신호를 보낼지 받을지)
Address bus (CPU가 메모리에게 보내는 주소)
Data Bus (메모리가 주소에 있는 데이터를 전달하거나, CPU가 보내는 데이터를 저장)

CPU가 메모리에서 필요한 것들을 가져다가 연산하고, 연산 결과를 다시 메모리에 돌려주어 저장
더 빠르게 전달하기 위해 자주 쓰이는 것들은 캐시 메모리에 저장
C, C++로 프로그램을 짜는 게 다른 언어에 비해 속도가 훨씬 빠름

[CPU가 사용하는 언어]
이진수로 되어있는 기계어
이진수는 사람이 이해하기 어렵기 때문에 프로그래밍을 할 땐 사람에게 더 가까운 언어를 사용하고, 그후 이진수로 다시 변환해서 컴퓨터가 이해함
- 어셈블리 언어 (니모닉 Mnemonic) : LOAD, ADD, STORE 등
- 고수준 프로그래밍 언어

[CPU의 기본 구조]
Control Unit (제어 장치)
ALU, Arithmetic Logic Unit (산술 논리 장치)
레지스터들

- PC (프로그램 카운터) : 다음 명령어의 메모리 주소 저장
- CIR (명령어 레지스터) : 메모리에서 읽어온 명령어 저장
- MAR (메모리 주소 레지스터) : 읽거나 쓸 메모리 주소 저장
- MDR (메모리 데이터 레지스터) : 메모리에서 읽어온 데이터 저장
- Accumulator (어큐뮬레이터) : 연산에 사용되는 데이터 저장

[정보의 단위]
bit (Binary digt, 비트)
- 정보의 기본 단위
- 0 또는 1
1 byte (바이트) = 8 bit
- 8개의 비트를 묶어서 1바이트라고 함
- 8비트인 이유는, 컴퓨터 발달 과정에서 관습적으로 굳어진 숫자
16 bit Word, 32 bit Word, 64 bit word
- CPU가 데이터를 다루는 기본 단위
- 1워드는 16비트일수도, 32비트일 수도, 64비트일 수도 있음
- 2진수 주소의 길이

[부호가 있는 8비트 정수]
2의 보수
1. 2진수를 보수(0은 1로, 1은 0으로)
2. 맨 뒤에 더하기 1
첫 비트가 1이면 음수, 0이면 양수

[8비트 정수의 범위]
부호가 있는 Signed 정수
- 10진수 : -123 ~ 127 (총 256개)
- 2진수 : 10000000 ~ 01111111

[부호가 없는 Unsigned 정수]
- 10진수 : 0 ~ 255 (총 256개)
- 2진수 : 00000000 ~ 11111111
